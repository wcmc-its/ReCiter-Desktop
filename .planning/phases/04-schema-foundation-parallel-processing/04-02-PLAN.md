<objective>
Refactor the pipeline to yield SSE events in true completion order and use adaptive worker counts, then update the frontend to handle the new event flow.

Purpose: The current pipeline awaits researchers in submission order, meaning a fast researcher queued behind a slow one appears blocked in the UI. This plan switches to `asyncio.as_completed` so users see real-time progress, and adapts worker count based on PubMed API key availability (8 with key, 3 without) for better throughput.

Output: Refactored pipeline_runner.py with as_completed + adaptive workers, updated router creating PipelineRun rows, updated frontend removing `processing` event and showing worker count.
</objective>
