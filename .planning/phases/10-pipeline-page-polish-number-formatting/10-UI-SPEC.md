## Interaction Contract

### ETA Countdown (PIPE-02 — locked per D-03, D-04, D-05)

**State:** `avgTimePerResearcher` (React state, number, milliseconds). `lastCompletionTimeRef` (React ref, `number | null`).

**Behavior:**
- ETA is hidden until at least one researcher completes (`completed > 0` gate — unchanged from current).
- On `started` SSE event: initialize `lastCompletionTimeRef.current = Date.now()`.
- On `complete_one` SSE event: compute `delta = Date.now() - lastCompletionTimeRef.current`. Update `avgTimePerResearcher` as simple moving average: `prev === 0 ? delta : (prev + delta) / 2`. Set `lastCompletionTimeRef.current = Date.now()`.
- ETA display expression unchanged: `formatDuration(avgTimePerResearcher * (total - completed))`. ETA decreases as completions arrive; freezes between completions; never counts up.
- Remove: the `setStartTime(prevStart => { setAvgTimePerResearcher(elapsed / completedCount) })` block in `complete_one` handler.

**What does NOT change:** `startTime` state (still used for "Elapsed:" display). `formatDuration()` function. The ETA render expression.

### Progress Bar Animation (PIPE-03 — locked per D-06)

**Current (wrong):** `from { background-position: 40px 0; } to { background-position: 0 0; }` — stripes move right-to-left.

**Fixed:** `from { background-position: 0 0; } to { background-position: 40px 0; }` — stripes move left-to-right (forward direction).

**File:** `frontend/app/globals.css`, `@keyframes pipeline-stripe` block (lines 149–152). No other changes to `.pipeline-progress-animated`.

### Status Column Font Fix (PIPE-04 — locked per D-07)

**File:** `frontend/components/pipeline-row.tsx`, line 139.

**Current:** `<span className={\`text-sm flex items-center gap-2 ${PHASE_COLORS[phase]}\`}>`

If the status column still appears misaligned relative to the header, adjust the `1.2fr` fraction in `grid-cols-[1.5fr_1fr_0.6fr_1.2fr_1fr]` at implementor's discretion. This is a Claude's Discretion call — do not change proportions preemptively.

### Bottleneck Detection (PIPE-05 — verified per D-08, D-09)

No code change required. The `isBottleneck` conditional (`avgTimePerResearcher > 0 && elapsed > avgTimePerResearcher * 2`) is structurally correct. The false-positive behavior is a symptom of the cumulative ETA bug. Fixing PIPE-02 resolves PIPE-05 as a side-effect. Verify after PIPE-02 is implemented.

### Number Formatting (FMT-01 — locked per D-10, D-11)

Use inline `.toLocaleString()` at each call site. No utility wrapper. No locale or options arguments.

**Exact changes (3 sites):**

| File | Line | Current | Fixed |
|------|------|---------|-------|
| `frontend/app/pipeline/page.tsx` | ~348 | `{totalArticles}` | `{totalArticles.toLocaleString()}` |
| `frontend/app/pipeline/page.tsx` | ~349 | `{totalScored}` | `{totalScored.toLocaleString()}` |
| `frontend/components/pipeline-row.tsx` | ~138 | `{articleCount ?? "\u2014"}` | `{articleCount != null ? articleCount.toLocaleString() : "\u2014"}` |

**Note on null pattern:** `articleCount != null` (loose inequality) catches both `null` and `undefined`. `0` is not null — `(0).toLocaleString()` returns `"0"` correctly. The `?? "—"` pattern must be replaced entirely; it cannot be combined with `.toLocaleString()`.

**All other numeric displays confirmed already using `.toLocaleString()`:** completion summary (`totalScored.toLocaleString()`), confidence tier cards (`summary.high_confidence.toLocaleString()`, etc.), dashboard, stats page, articles page. No changes needed elsewhere.
